# PaperForge-Agent 小范围测试计划

本测试计划用于产品化版本上线前后的小范围课题组试用。试用目标是验证单项目 Agent 研究闭环是否稳定、可理解、可审计。

## 试用范围

- 试用人数：10-15 名课题组成员。
- 角色：1 名维护者，若干博士生/硕士生/实习生研究用户。
- 时间窗口：建议 5-7 天，避免长期开放导致反馈失焦。
- 项目类型：博弈论、平台经济学、双边市场、Hotelling、机制设计或比较静态分析相关主题。

## 上线前必须完成

- `npm run lint`
- `npm test`
- `npx tsc --noEmit`
- `npm run build`
- 浏览器冒烟测试。
- 一次完整 demo 从 idea 跑到 paper patch。
- 导出并检查项目审计报告不含密钥。

## 成功指标

- 至少 3 次完整 idea-to-paper-patch 运行。
- 刷新页面后没有项目数据丢失。
- 所有核心资产变更都以 reviewable patch 出现。
- 用户能区分哪些数学声明已验证、哪些需要人工复核。
- 用户能说清“需修正”“条件不足”“人工复核”三种提示分别该怎么处理。
- 每个试用项目都能导出审计报告。
- 至少 70% 试用者能指出“下一步应该做什么”和“为什么停在这里”。

## 停止条件

出现任一情况立即暂停试用：

- 项目数据无法保存或恢复。
- patch 应用后改变了错误资产。
- failed math verification 被显示为 passed。
- “需修正”或“人工复核”只显示计数，没有处理说明或入口，导致用户无法继续。
- provider secrets、Clerk secrets、Tavily MCP URL 或私密 key 出现在 UI、日志或导出报告中。
- 待审核 patch 被自动绕过并写入正式资产。
- 论文预览中已有 `$$...$$` 块级公式被二次包裹或渲染成破碎文本。

## 测试任务

每位用户至少完成：

1. 新建一个研究项目。
2. 打开联网搜索并生成方向。
3. 选择一个方向并审阅模型 patch。
4. 应用或拒绝模型 patch。
5. 若应用模型，继续生成均衡和性质分析。
6. 查看数学验证摘要。
7. 生成论文草稿或章节级改写建议。
8. 打开论文预览，检查公式块、行内公式和代码块是否正常渲染。
9. 导出项目审计报告。
10. 提交反馈模板。

## 验收结论

小范围测试结束后，将结果分为三类：

- 可以继续扩大试用：自动检查通过，无停止条件，核心路径可被多数用户理解。
- 需要修复后再试：无数据或安全事故，但用户在关键路径中频繁卡住。
- 暂停上线：触发停止条件，或无法保证 patch 审核、数据保存、密钥安全。
