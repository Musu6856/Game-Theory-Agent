# PaperForge-Agent 课题组试用指南

这份指南面向小范围课题组用户。PaperForge-Agent 不是自动替你完成论文的系统，而是一个可审阅、可暂停、可回放的理论研究 Agent。它适合把一个初步博弈论研究想法推进到方向候选、模型候选、符号均衡候选、比较静态候选和论文草稿补丁。

## 适合做什么

- 从一句中文研究想法出发，生成可建模的研究方向。
- 在用户允许时进行联网搜索，并把结果整理成“来源依据”。
- 基于已选方向生成模型候选，并把模型变更放入“待审核修改”。
- 在模型确认后生成均衡候选、性质分析候选和论文草稿候选。
- 对单个论文章节生成“章节改写建议”，但应用前不会覆盖原文。
- 导出项目审计报告，用于回看来源、Agent 执行记录、数学验证和资产版本历史。

## 必须人工审阅什么

- 研究方向是否真的值得做，Agent 不替用户选择方向。
- 模型参与方、时序、效用函数、利润函数和参数假设是否合理。
- 符号均衡推导是否符合课题组的理论标准。
- 比较静态命题是否有经济直觉，是否只是形式上可导。
- 论文草稿是否符合真实写作规范、引用格式和学术表达。
- 所有“待审核修改”都需要用户应用或拒绝后，才会进入正式资产。

## 关键界面词汇

- “联网搜索”：系统调用公开学术或网页搜索服务来辅助方向发现。
- “来源依据”：系统保存的来源标题、URL、检索时间、摘要和相关性说明。
- “联网来源”：来源页里展示的公开网页、论文元数据或行业/政策资料。
- “待审核修改”：Agent 提出的 reviewable patch，应用前不会改正式资产。
- “数学验证”：系统对符号来源、简单偏导、符号方向和条件强弱的结构化检查。
- “人工复核”：当前自动验证不支持或不应强行判断，需要用户自己检查。

## 推荐试用流程

1. 输入一个明确但不必完整的研究想法，例如“二手平台佣金与补贴如何影响买卖双方参与”。
2. 打开“联网搜索”，生成研究方向。
3. 阅读每个方向的来源依据，选择一个方向进入建模。
4. 审阅模型 patch。重点看参与方、平台集合、时序、效用函数、利润函数、假设和符号表。
5. 应用模型 patch 后确认模型，再生成符号均衡。
6. 审阅均衡 patch。重点看 FOC、闭式解、存在条件和 warnings。
7. 应用均衡 patch 后生成性质分析。
8. 阅读数学验证摘要。若出现“需要修正”或“条件不足”，先修正或重新生成，不建议直接写论文。
9. 应用性质分析 patch 后生成论文草稿或对单个章节使用“章节改写建议”。
10. 导出项目审计报告，作为试用反馈附件。

## 当前能力边界

- 目前是单项目工作流，不包含跨项目记忆。
- 当前长任务续跑是步骤级重试，不恢复半个 HTTP 请求。
- 数学验证覆盖简单线性表达式、简单分式、连等式和部分符号条件，不等于完整 CAS 证明。
- 暂不自动应用模型、均衡、性质分析或论文 patch。
- 联网来源仅用于方向和背景依据，正式论文引用格式仍需人工补齐。
