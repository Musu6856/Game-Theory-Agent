# PaperForge-Agent Demo 场景

这些场景用于上线前演示和小范围试用培训。每个场景都应完整跑到至少一个待审核 patch，并导出项目审计报告。

## 场景一：二手平台佣金与补贴

Prompt：

```text
我想研究二手交易平台上，平台对卖家收取佣金、对买家提供补贴时，买卖双方的加入决策和平台利润如何变化。希望用 Hotelling 或双边市场模型分析。
```

预期 checkpoints：

- 方向发现：生成多个可建模方向，并展示联网来源或开放学术来源。
- 模型生成：提出平台、买家、卖家、佣金、补贴和跨边参与的模型 patch。
- 均衡求解：生成 FOC、闭式解或说明符号求解限制。
- 性质分析：给出佣金、补贴或匹配效率相关的比较静态候选。
- 论文输出：生成整体论文草稿或对模型章节生成章节级改写 patch。

预期 patches：

- `model` patch：模型设定。
- `equilibrium` patch：符号均衡。
- `properties` patch：比较静态命题。
- `paper` patch：论文草稿或单章节改写。

已知限制：

- 若均衡表达式过复杂，数学验证可能转为人工复核。
- 正式引用需要试用者自己补齐格式。

## 场景二：平台算法推荐与商家质量投资

Prompt：

```text
我想研究电商平台的算法推荐强度如何影响商家的质量投资。平台可以提高推荐精准度，但商家也可能因为流量分配规则改变而调整质量投入。
```

预期 checkpoints：

- 方向发现：区分推荐机制、商家质量投资和平台定价/排序规则。
- 模型生成：检查是否明确平台、商家、消费者、推荐强度和质量成本。
- 均衡求解：关注质量投资 FOC 与推荐强度参数。
- 性质分析：检查推荐强度对质量、消费者福利或平台利润的方向。

预期 patches：

- 模型 patch 应包含推荐强度参数和质量成本假设。
- 性质 patch 可能出现“条件不足”，需看是否缺少成本凸性或参数正负假设。

已知限制：

- 如果模型把推荐系统写得过经验化，可能不适合当前符号均衡链路。
- 多商家异质性可能需要人工简化为两商家或代表性商家。

## 场景三：会员制平台与广告干扰

Prompt：

```text
我想研究内容平台在会员订阅和广告收入之间的权衡：提高广告强度会增加广告收入，但会降低用户体验并影响订阅转化。
```

预期 checkpoints：

- 方向发现：找到会员订阅、广告干扰、双边平台或媒体经济学相关来源。
- 模型生成：明确用户效用、广告商收益、平台订阅费和广告强度。
- 均衡求解：检查订阅费和广告强度的最优条件。
- 性质分析：关注广告干扰成本、广告需求弹性对订阅费和广告强度的影响。
- 论文章节改写：对“引言与研究问题”生成更清晰的动机段落。

预期 patches：

- `model` patch：订阅费、广告强度、用户体验损失。
- `equilibrium` patch：平台最优订阅费/广告强度。
- `paper` section patch：引言或模型章节单独改写。

已知限制：

- 广告市场和订阅市场同时内生时，完整闭式解可能太复杂。
- 当前系统更适合先形成可审阅理论草稿，而不是直接得到最终论文结论。
